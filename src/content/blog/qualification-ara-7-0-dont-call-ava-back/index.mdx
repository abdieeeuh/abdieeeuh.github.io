# Dont call ava back

## Intro

The title sounds like a joke, and the challenge description leans into that. But the binary itself is much more interesting than the story around it.

This is a self-modifying Windows PE. Instead of keeping the real verification logic in one readable function, the program patches its own validator at runtime and redirects execution into a blob stored in `.data`.

So the real solve path is:

1. find the patcher
2. extract the verifier blob
3. recover the math model hidden in that blob
4. solve the input that makes the verifier accept

## The Patcher: The Verifier Is Rewritten at Runtime

The important helper lives at `0x1400080b0`. Its behavior is the real key to the challenge.

Relevant disassembly:

```bash
$ r2 -q -e scr.color=false -e bin.relocs.apply=true -c "aa; af patcher 0x1400080b0; pdf @ patcher" chall.exe | head -n 90
┌ 264: patcher (int64_t arg2);
│           0x1400080f1      lea rdi, [fcn.1400010e0]   ; 0x1400010e0
│           0x140008101      call qword [sym.imp.KERNEL32.dll_VirtualProtect]
│           0x14000811a      call qword [sym.imp.KERNEL32.dll_VirtualAlloc]
│           0x140008126      lea rdx, [0x14000c080]
│           0x140008133      call sub.VCRUNTIME140.dll_memmove
│           0x140008138      mov word [rdi], 0xb848
│           0x140008190      mov word [rdi + 0xa], 0xe0ff
│           0x14000819b      call qword [sym.imp.KERNEL32.dll_VirtualProtect]
└           0x1400081af      ret
```

This means the program does all of the following:

- changes the first bytes of `fcn.1400010e0` to writable
- allocates a fresh executable buffer with `VirtualAlloc`
- copies `0x4b0` bytes from `.data` at `0x14000c080`
- writes a 12-byte stub:
  - `48 b8 <imm64>` -> `mov rax, <blob_ptr>`
  - `ff e0` -> `jmp rax`

So the original validator becomes only a launcher. The actual verification logic lives in the blob at `.data:0x14000c080`.

Cross-reference confirmation:

```bash
$ r2 -q -e scr.color=false -e bin.relocs.apply=true -c "aa; aar; axt @ 0x14000c080" chall.exe
(nofunc) 0x140008126 [DATA:r--] lea rdx, [0x14000c080]
```

From this point on, the right target is not `fcn.1400010e0` anymore. It is the copied blob.

## Inside the Verifier Blob

### 1. The input length must be exactly 56

The start of the blob walks the string and compares the final length against `0x38`:

```bash
$ r2 -q -e scr.color=false -e bin.relocs.apply=true -e bin.strings=false -e anal.strings=false -c "aa; s 0x14000c0ad; pd 14" chall.exe
0x14000c0ad      xor edx, edx
0x14000c0af      mov r9, rcx
0x14000c0b2      mov r8d, edx
0x14000c0b5      cmp byte [rcx], dl
0x14000c0b7      je 0x14000c4f5
0x14000c0c0      inc r8d
0x14000c0c7      cmp byte [rax], dl
0x14000c0c9      jne 0x14000c0c0
0x14000c0cb      cmp r8d, 0x38
0x14000c0cf      jne 0x14000c4f5
```

So the candidate flag length is fixed at 56 bytes.

### 2. The blob converts bytes into float32 values

Later, the code loads groups of eight bytes and turns them into float vectors:

```bash
$ r2 -q -e scr.color=false -e bin.relocs.apply=true -e bin.strings=false -e anal.strings=false -c "aa; s 0x14000c3bb; pd 12" chall.exe
0x14000c3bb      vpmovzxbd ymm1, qword [r9]
0x14000c3c8      vcvtdq2ps ymm2, ymm1
0x14000c3cc      vpmovzxbd ymm1, qword [r9 + 8]
0x14000c3d7      vfmadd231ps ymm9, ymm2, ymm8
0x14000c3e6      vfmadd231ps ymm0, ymm9, ymm9
```

Conceptually:

- each input byte becomes an integer in `0..255`
- those integers are converted into `float32`
- the verifier applies fused multiply-add operations over them

### 3. Success means `abs(sum) < 0.5`

The final decision is surprisingly clean:

```bash
$ r2 -q -e scr.color=false -e bin.relocs.apply=true -e bin.strings=false -e anal.strings=false -c "aa; s 0x14000c4cf; pd 14" chall.exe
0x14000c4cf      vmovd eax, xmm2
0x14000c4d3      btr eax, 0x1f
0x14000c4d7      mov dword [rbp], 0x3f000000
0x14000c4de      vmovss xmm0, dword [rbp]
0x14000c4e3      vaddss xmm2, xmm0, dword [rbp + 4]
0x14000c4e8      vmovd xmm1, eax
0x14000c4ec      vcomiss xmm2, xmm1
0x14000c4f0      seta al
```

That translates to:

```c
return (0.5f > fabsf(sum));
```

This is a very helpful observation, because the verifier does not require an exact symbolic match. It only requires the final floating-point score to be small enough.

## Recovering the Mathematical Model

The blob writes a large number of constants onto the stack using `mov dword [rbp+disp], imm32`.

One important constant appears right at the start:

```asm
mov dword [rbp], 0xdeadbeef
```

That value is used as an XOR mask before the immediate values are reinterpreted as `float32`.

Once the constants are grouped correctly, the verifier becomes:

```c
dead = 0xdeadbeef;
bias[j]   = as_float(dead ^ raw_bias_u32[j]);
weight[j] = as_float(dead ^ raw_weight_u32[j]);
K[i]      = as_float(dead ^ raw_const_u32[i]);
```

And the verification logic itself reduces to:

```c
for (int i = 0; i < 56; i++) {
  int j = i % 8;
  float x = (float)(uint8_t)flag[i];
  float y = x * weight[j] + bias[j];
  sum += (y * y) + K[i];
}

return fabsf(sum) < 0.5f;
```

This is the key simplification.

Each position depends only on:

- the byte at that position
- one of eight repeating weights/biases
- one per-position constant `K[i]`

So instead of solving one huge coupled problem, we can solve each byte almost independently by pushing each term close to zero.

If:

```text
term = y^2 + K[i]
```

then the target is:

```text
y^2 ~= -K[i]
y ~= +/-sqrt(-K[i])
x ~= (+/-sqrt(-K[i]) - bias[j]) / weight[j]
```

That gives a small search window around the real-valued solution for each byte.

## Solver Script

I attached the archived solver as [solve.py](./solve.py). The full script is included below.

```python
#!/usr/bin/env python3
import math
import struct
from dataclasses import dataclass
from typing import Dict, List

import numpy as np
from capstone import Cs, CS_ARCH_X86, CS_MODE_64
from capstone.x86 import X86_OP_IMM, X86_OP_MEM, X86_REG_RBP


@dataclass(frozen=True)
class Model:
    bias: np.ndarray
    weight: np.ndarray
    k: np.ndarray


def u32_to_f32(u: int) -> np.float32:
    """Reinterpret 32-bit int bits as float32."""
    return np.frombuffer(struct.pack("<I", u & 0xFFFFFFFF), dtype=np.float32)[0]


def fma32(a: np.float32, b: np.float32, c: np.float32) -> np.float32:
    """
    Approximate float32 FMA.
    CPU instr is fused (single rounding). For this challenge, non-fused
    arithmetic is still good enough because the threshold is abs(sum) < 0.5.
    """
    return np.float32(np.float64(a) * np.float64(b) + np.float64(c))


def extract_stores(exe_path: str, blob_off: int, blob_len: int, blob_va: int) -> Dict[int, int]:
    """
    Extract instructions of the form:
      mov dword [rbp + disp], imm32
    and keep the first write for each displacement.
    """
    with open(exe_path, "rb") as f:
        f.seek(blob_off)
        blob = f.read(blob_len)

    md = Cs(CS_ARCH_X86, CS_MODE_64)
    md.detail = True

    first_write: Dict[int, int] = {}
    for ins in md.disasm(blob, blob_va):
        if ins.mnemonic != "mov" or len(ins.operands) != 2:
            continue

        dst, src = ins.operands
        if dst.type != X86_OP_MEM or src.type != X86_OP_IMM:
            continue

        mem = dst.mem
        if mem.base != X86_REG_RBP or mem.index != 0:
            continue

        if dst.size != 4:
            continue

        disp = mem.disp
        if disp > 0x7FFFFFFFFFFF:
            disp -= 1 << 64
        if disp < 0:
            continue

        if disp not in first_write:
            first_write[disp] = src.imm & 0xFFFFFFFF

    return first_write


def build_model(first_write: Dict[int, int]) -> Model:
    """
    Constant layout recovered from the blob:
      [rbp + 0x00] = dead XOR key (first write)
      [rbp + 0x08 .. 0x24] = raw bias u32 values
      [rbp + 0x28 .. 0x44] = raw weight u32 values
      [rbp + 0x50 .. 0x12C] = raw K u32 values
    """
    dead = first_write[0x00]
    if dead != 0xDEADBEEF:
        raise ValueError(f"unexpected dead key: {dead:#x} (expected 0xdeadbeef)")

    bias = np.array([u32_to_f32(dead ^ first_write[0x08 + 4 * j]) for j in range(8)], dtype=np.float32)
    weight = np.array([u32_to_f32(dead ^ first_write[0x28 + 4 * j]) for j in range(8)], dtype=np.float32)
    k = np.array([u32_to_f32(dead ^ first_write[0x50 + 4 * i]) for i in range(56)], dtype=np.float32)

    return Model(bias=bias, weight=weight, k=k)


def term(model: Model, byte_val: int, pos: int) -> float:
    """Compute term_i = (x*w + b)^2 + K[i] using float32 operations."""
    j = pos % 8
    x = np.float32(byte_val)
    y = fma32(x, model.weight[j], model.bias[j])
    t = fma32(y, y, model.k[pos])
    return float(t)


def solve(model: Model) -> bytes:
    """
    For each position i:
      find x in 0..255 that makes term(i) as close to zero as possible.

    Since term = y^2 + K, the target is y ~= +/-sqrt(-K).
    """
    out: List[int] = []

    for i in range(56):
        j = i % 8
        k = float(model.k[i])

        target = math.sqrt(-k)
        candidates = set()

        for sgn in (+1.0, -1.0):
            x_real = (sgn * target - float(model.bias[j])) / float(model.weight[j])
            lo = int(math.floor(x_real)) - 3
            hi = int(math.ceil(x_real)) + 3
            for b in range(lo, hi + 1):
                if 0 <= b <= 255:
                    candidates.add(b)

        best = min(candidates, key=lambda b: abs(term(model, b, i)))
        out.append(best)

    return bytes(out)


def main() -> None:
    exe_path = "chall.exe"
    blob_va = 0x14000C080
    blob_off = 0x9800 + 0x80
    blob_len = 0x4B0

    first_write = extract_stores(exe_path, blob_off, blob_len, blob_va)
    model = build_model(first_write)

    flag = solve(model)
    total = sum(term(model, flag[i], i) for i in range(56))

    print(flag.decode("ascii"))
    print(f"[check] sum={total:.6f}  abs(sum) < 0.5 => {abs(total) < 0.5}")


if __name__ == "__main__":
    main()
```

## Pseudocode Summary

A readable version of the verifier looks like this:

```c
bool verify(const char *s) {
  if (strlen(s) != 56) return false;

  float sum = 0.0f;
  for (int i = 0; i < 56; i++) {
    int j = i % 8;
    float x = (float)(uint8_t)s[i];
    float y = x * weight[j] + bias[j];
    sum += (y * y) + K[i];
  }

  return fabsf(sum) < 0.5f;
}
```

And the patching logic is essentially:

```c
void patch_validator() {
  void *blob = VirtualAlloc(NULL, 0x4b0, MEM_COMMIT | MEM_RESERVE, PAGE_EXECUTE_READWRITE);
  memmove(blob, (void *)0x14000c080, 0x4b0);
  write_stub((void *)0x1400010e0, blob);   // mov rax, blob ; jmp rax
}
```

## Closing

The AVX and self-modifying behavior make this challenge look much harder than the final model really is.

The important lesson is not "understand every vector instruction immediately". It is:

- find the runtime patcher
- isolate the hidden logic
- recover the constants
- reduce the verifier into math you can solve directly

Once the blob was separated from the wrapper, the whole problem became much more mechanical.
